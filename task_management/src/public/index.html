Mengapa kami menggunakan Docker?

Docker merupakan teknik virtualisasi namun tingkat Operating System, berbagi kernel yang sama sehingga lebih cepat dan lightweight. Layanan yang menyediakan kemampuan untuk mengemas dan menjalankan sebuah aplikasi dalam sebuah lingkungan yang terisolasi yang disebut container. Memiliki Docker Container yang merupakan paket software berisi semua dependensi yang diperlukan untuk menjalankan aplikasi tertentu. Semua konfigurasi dan instruksi untuk memulai atau menghentikan container diperintahkan oleh sebuah komponen yang disebut image docker. Pengelolaan Container juga mudah berkat bantuan Docker API atau CLI, apabila beberapa container diperlukan user bisa mengontrolnya dengan Docker Compose Tool.

Bagaimana cara kerja Docker?

Client Docker Komponen utama untuk membuat, mengelola, dan menjalankan aplikasi dalam container serta metode utama untuk mengontrol server Docker melalui CLI seperti Command Prompt atau Terminal. Server Docker (Daemon Docker) Server Docker menunggu permintaan Rest API yang dibuat oleh client Docker serta mengelola image dan container. Image Docker Komponen yang memerintahkan server Docker terkait persyaratan tentang cara container Docker dibuat. Registry Docker Aplikasi sisi server open-source yang digunakan untuk menghosting dan mendistribusikan image Docker. Registry berguna untuk menyimpan image secara lokal dan mengelolanya sepenuhnya.
